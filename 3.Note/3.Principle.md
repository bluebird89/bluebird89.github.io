# 原则

- 单一职责：一个类应该只做一类事情 一个方法最多只做一件事情
- 不要做超前的设计
- 第三次重复它的时候才抽出为方法或封装类

# 逻辑

拆分逻辑变成步骤的注释； 将注释视复杂度变成方法； 递归对方法使用此技巧。

功能拆分颗粒度，并做架构
